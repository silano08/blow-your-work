종합 평가
TaskWave는 팀원의 Todo를 회사 목표와 이니셔티브에 자동 매핑하고, 팀 현황·목표 관리·AI 감사 흐름을 하나의 대시보드로 묶어 생산성 문제에 잘 맞춘 제출물입니다. GitHub Copilot SDK가 핵심 분석 경로에 실제로 사용되며, 구조화된 JSON 출력·신뢰도·한국어 근거를 요구하는 프롬프트 설계도 도메인에 적합합니다. 로그인 없이 바로 주요 화면을 탐색할 수 있고, 신뢰도 배지, 빈 상태, 토스트, AI 감사 탭 등 사용자 흐름도 비교적 매끄럽습니다. Azure Container Apps 배포, ACR/GitHub Actions 기반 파이프라인, `/health`와 `/ready` 같은 운영 기본기도 갖추고 있습니다.

다만 완성도와 신뢰성 측면에서는 아직 보완이 필요합니다. 핵심 AI 분석 경로에 스키마 불일치로 새 Todo 분석이 실패할 가능성이 있고, 외부 AI 호출의 재시도·로깅·폴백·테스트가 제한적입니다. Azure OpenAI는 주 분석 계층이라기보다 선택적 폴백에 가깝고, 라이브 환경에서는 환경변수 미설정으로 규칙형 응답이 반환되는 것으로 보입니다. 또한 AI 감사/설정 API가 인증 없이 공개되어 의사결정 로그 조회와 리뷰 상태 변경이 가능한 점은 보안·신뢰 관점에서 큰 약점입니다. Key Vault, Managed Identity, App Insights/Log Analytics, OAuth `state` 검증, 보안 쿠키, CSP/HSTS 같은 클라우드 네이티브 보안·관측성 구성도 부족합니다.

가장 영향력 있는 개선은 핵심 AI 분석과 감사 흐름을 “운영 가능한 신뢰 계층”으로 강화하는 것입니다. 구체적으로는 Todo 분석 스키마를 정합화하고 실패 시 재시도·명확한 오류 표시·안전한 폴백을 추가하며, AI 감사/설정 API에 인증과 역할 기반 권한을 적용해야 합니다. 여기에 실제 Azure OpenAI 설정, Key Vault/Managed Identity, 관측성, 테스트를 연결하면 현재의 설득력 있는 데모가 실제 팀에서 안전하게 사용할 수 있는 제품 수준으로 크게 올라갈 것입니다.

Effective Use of Copilot SDK
4/5
점수는 4점입니다. GitHub Copilot SDK가 단순 장식이 아니라 Todo를 회사 목표/이니셔티브에 자동 매핑하는 핵심 가치에 직접 연결되어 있습니다. 프롬프트는 도메인 용어와 구조화된 JSON 출력, 신뢰도, 한국어 근거를 요구하도록 설계되어 있고, 실제 `/analysis/trigger` 경로에서 DB의 활성 전제와 미분석 Todo를 모아 Copilot 세션 이벤트로 응답을 수집한 뒤 결과를 저장합니다. 다만 도구 호출은 실질적으로 없고, 재시도/컨텍스트 윈도우 관리/감사 큐와의 깊은 연결은 제한적이며, 제출 설명의 Azure fallback은 Copilot 분석 경로에는 보이지 않아 5점 수준의 정교한 에이전트 설계까지는 아닙니다.

Productivity Impact & Problem Fit
4/5
명확한 사용자(팀장, 팀원, 경영진)와 실제 생산성 문제(목표 단절, 데일리 스크럼 피로, 팀 간 블로커 불투명성)에 잘 맞는 제출물입니다. Todo를 회사 목표에 자동 매핑하고, 팀 현황·AI 감사·목표 관리를 한 대시보드에서 보게 하려는 흐름은 회의와 상태 파악 비용을 줄이는 설득력 있는 방향입니다. 다만 효과가 실제로 얼마나 절감되는지에 대한 측정 자료나 전후 비교 아티팩트는 부족하고, PRD 자체도 일부 핵심인 비동기 스크럼 저장·공유를 다음 버전 후보로 남겨 두어 5점 수준의 압도적 입증까지는 아닙니다.

Azure AI & Cloud Integration
3/5
Azure Container Apps에 실제 배포되어 동작하고 ACR/GitHub Actions 기반 배포 파이프라인과 헬스체크는 확인됩니다. 다만 AI의 주 분석 경로는 GitHub Copilot SDK이고, Azure OpenAI는 코드상 선택적 폴백/추천 서비스로만 구현되어 있으며 라이브 환경에서는 해당 환경변수가 배포되지 않아 규칙형 폴백이 반환되는 것으로 보입니다. IaC, managed identity, Key Vault, App Insights/Log Analytics 같은 클라우드 네이티브 보안·관측성 구성도 확인되지 않았습니다. 따라서 “배포된 Azure 관리 서비스는 있으나 Azure AI 계층의 실제 운영 근거가 약한” 3점으로 평가합니다.

Functionality & Technical Execution
3/5
데모 기준으로는 앱이 실제 배포 환경에서 끝까지 열리고 주요 화면까지 진입하므로 ‘동작함’ 수준은 충족합니다. FastAPI 라우터 분리, SQLite 초기화/시드, `/health`/`/ready`, Docker 및 Azure 배포 워크플로가 갖춰져 있어 기본 기술 실행은 양호합니다. 다만 핵심인 실제 AI 분석 경로에 스키마 불일치가 있어 새 Todo 분석이 실패할 가능성이 높고, 관리자/팀장 전용 기능의 권한 검증이 부족하며, 외부 AI 호출의 재시도·로깅·폴백이 제한적이고 테스트가 보이지 않습니다. 따라서 안정적·견고한 4점보다는 핵심 기능이 데모/시드 데이터 중심으로 작동하는 3점이 적절합니다.

User Experience & Workflow Design
4/5
전반적으로 낮은 마찰의 데모 진입, 명확한 정보 구조, 반응형 내비게이션, 토스트/빈 상태/신뢰도 배지/AI 감사 흐름이 잘 갖춰져 있어 ‘Smooth’ 수준입니다. 사용자는 로그인 없이 바로 대시보드에 들어가 할일 추가, AI 분석, 팀 현황, 목표 관리, AI 감사까지 자연스럽게 이동할 수 있습니다. 다만 AI 작업 중 취소·재시도·버튼 비활성화 같은 사용자 제어가 제한적이고, 오류를 조용히 무시하는 코드가 많으며, 체크박스/아이콘 버튼이 실제 접근성 역할이나 키보드 조작을 충분히 제공하지 않아 ‘Delightful’까지는 어렵습니다.

Responsible AI, Security & Trust
3/5
기본적인 책임 있는 AI 요소는 구현되어 있습니다. AI 결과가 신뢰도 배지와 근거로 식별되고, `AI 감사` 탭에서 승인·이의제기·수정하는 HITL 흐름이 있으며, 실제 배포도 동작했습니다. 또한 실제 시크릿 값은 저장소에서 발견하지 못했고 환경변수/GitHub Secrets 사용 의도가 보입니다. 다만 보안·신뢰 관점에서 중요한 결함이 있어 4점 이상은 어렵습니다. 특히 AI 감사/설정 API가 인증 없이 공개되어 배포 URL에서 의사결정 로그가 그대로 조회되고, 누구나 리뷰 상태를 바꿀 수 있는 구조입니다. Key Vault/Managed Identity는 없고, OAuth `state` 검증·보안 쿠키 옵션·CSP/HSTS 설정도 부족합니다. 프롬프트 인젝션 방어는 JSON 출력 지시와 파싱 정도에 머물며, `PermissionHandler.approve_all` 사용은 도구 권한 방어 측면에서 위험합니다.

Innovation & Originality
4/5
이 제출물은 단순한 문서 채팅·요약·티켓 분류 봇을 넘어, 팀원의 일일 할일을 이니셔티브/목표와 자동 연결하고 그 결과를 조직 단위 그래프·히트맵·HITL 감사 흐름으로 보여주는 생산성 도구를 만들었다는 점에서 ‘발명적’입니다. 다만 핵심 AI는 주로 단일 프롬프트 기반 분류와 추천이며, 실제 학습형 피드백 루프나 독특한 에이전트 오케스트레이션까지는 보이지 않아 ‘획기적’ 단계에는 못 미칩니다.
