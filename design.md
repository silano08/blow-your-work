# Hecto Financial Design System

## Overview

Hecto Financial의 디자인 시스템은 금융 서비스에 필요한 신뢰감, 명확성, 반복 사용성을 중심에 둔다. 화면은 밝은 배경, 강한 텍스트 대비, 절제된 선과 면으로 구성하며, 중요한 행동은 지정된 primary 계열 토큰으로만 강조한다.

브랜드 인상은 과장된 장식보다 정확한 정보 전달과 안정적인 사용 흐름에서 만들어진다. 사용자는 결제, 인증, 조회, 입력처럼 실수가 민감한 업무를 수행하므로 모든 UI는 상태, 위계, 피드백을 즉시 이해할 수 있어야 한다.

브랜드 식별 색은 **Hecto Orange**이며 Overview 표지와 Color Semantic, 다수의 컴포넌트 강조(버튼 예시, 탭 인디케이터, 선택 상태, 프로그레스 바, 툴팁)에서 `hecto-orange-50/60`을 일관되게 사용한다. `primary-01/02`, `primary-text-01`, `button-primary`는 모두 Hecto Orange 계열에서 파생되며, 제품의 주요 행동과 브랜드 강조를 담당한다.

본 시스템은 **iOS 디바이스(375×812pt)** 기준으로 최적화되어 있으며, 안드로이드·태블릿·웹은 본 가이드를 응용해 사용한다.

Figma export 기준 구성 범위는 다음과 같다.

- **Foundation:** Grid, Spacing, Color(Palette·Semantic), Typography, Icon+Elements(Icon Grids, Radius, OS UI Kits, Face ID 포함)
- **Flexible Web Guide:** Desktop·Mobile typography, 다국어(KR/EN/JP/CN)
- **Component:** Icon, Avatar, Badge, Breadcrumbs, Buttons, Chip, Date Picker, Divide line, Dropdown, Loading, Navigation, Notice, Pagination, Progress Bar, Search, Selection Controls, Sheets, Stepper, Tab, Text Field, Toast Notification, Tooltip & Label

## Colors

색상은 Foundation의 Color Palette와 Color Semantic을 우선한다. 팔레트 색상은 원시 값이고, 실제 제품 화면에서는 semantic token을 먼저 사용한다.

- **Background:** `background-01`은 기본 흰 배경, `background-02`는 회색 면 배경.
- **Line:** `line-01`은 어두운 배경 위, `line-02`/`line-03`은 밝은 배경 위 구분선. 콘텐츠 분리보다 정보 묶음 정리 수준으로 사용한다.
- **Primary (Hecto Orange):** `primary-01`(#FFCFBD, HectoOrange-20), `primary-02`(#FF6114, HectoOrange-50), `primary-text-01`(#F85100, HectoOrange-60). 주요 행동, 브랜드 강조, 탭 인디케이터, 선택 활성, 프로그레스 진행 면에 사용한다.
- **Button:** `button-primary`는 `hecto-orange-60`(#F85100), `button-cta`는 `hecto-orange-50`(#FF6114)을 사용한다. 둘을 같은 영역에서 동시에 강조하지 말고, 한 화면의 가장 중요한 행동 하나에만 적용한다.
- **Text:** `text-01`(흰색, 어두운 배경 본문), `text-02`(`#FFFFFFCC`, 어두운 배경의 비강조 본문/메타), `text-03`(제목·본문 기본), `text-04`(보조 본문), `text-05`(설명·비활성 인접 보조), `text-06`(placeholder·강한 비활성).
- **Icon:** `icon-01`(어두운 배경), `icon-02`(밝은 배경 본문급), `icon-03`(보조), `icon-04/05`(비강조).
- **Status:** `status-danger`(#FF4242, Red-50)는 일반 오류·금지·삭제 표시에 사용한다. Text Field error border와 helper text 같은 입력 검증 경고도 `red-50`(#FF4242)을 사용하되, 필요한 경우 helper text나 icon 피드백을 함께 제공한다.

각 컬러 패밀리는 일반적으로 Base와 Text 단계가 지정되어 있다. 동일 가족 안에서 Base는 면/배경/포인트 색, Text는 같은 면 위에 올라가는 텍스트 색으로 사용한다.

| Palette | Base | Text |
|---|---|---|
| Hecto Orange | 50 (#FF6114) | 60 (#F85100) |
| Red | 50 (#FF4242) | 50 (#FF4242) |
| Green | 50 (#28C85D) | 70 (#187D3A) |
| Blue | 50 (#2465FF) | 60 (#1B4DF5) |
| Purple | 50 (#9A5BF1) | 60 (#833AE3) |

Neutral 50~70은 Neutral-5 배경 위에서 3:1 명도 대비, 70 이상은 4.5:1 명도 대비를 만족한다. 작은 본문 텍스트는 80 이상을 권장한다.

Neutral은 UI 구조의 대부분을 담당한다. Hecto Orange, Red, Green, Blue, Purple은 강조·상태·카테고리·보조 시각화에 사용하되 동일 화면에서 여러 강한 색이 경쟁하지 않게 한다.

## Typography

기본 서체는 한국어와 영어를 지원하는 **Pretendard**이다. 지원 weight는 Regular(400), Medium(500), SemiBold(600), Bold(700)이며 텍스트 목적과 기능에 맞춰 제한적으로 사용한다.

원본 Foundation Typography 표는 weight 컬럼을 라벨로 명시하지 않고 시각 예시로만 보여준다. 본 시스템은 Display·Heading은 Bold(700), Headline은 SemiBold(600), Body·Caption은 Regular(400)을 기본 weight로 고정하되, 강조가 필요할 때 한 단계 위 weight로 올린다.

Foundation typography는 앱/공통 UI의 기본 체계다. Display 1(48), Display 2(40), Heading 1(32), Heading 2(24), Headline 1(20), Headline 2(18), Body 1(16), Body 2(14), Body 3(13), Caption 1(12), Caption 2(11)을 사용한다.

Line-height는 일반 UI 텍스트에는 Normal 140%를 사용한다. 문장형 본문, 긴 설명, 약관, 안내문처럼 읽기 지속 시간이 긴 텍스트는 Reading 160%를 사용한다(`body-1/2-reading`, `caption-1-reading`). Letter spacing은 큰 제목 계열(Display·Heading·Headline) -3%, Body·Caption 계열 -2%를 따른다.

Flexible Web Guide는 웹 전용 typography를 별도로 둔다. Desktop은 Web-Display-1 52px부터 Web-Caption-2 11px까지, Mobile은 Web-Display-1 40px부터 Web-Caption-2 11px까지 사용한다. 반응형 웹에서 desktop token을 단순 축소하지 말고 mobile token으로 단계 전환한다. 본문보다 한 단계 낮은 설명 텍스트에는 `web-desktop-description-1`/`web-mobile-description-1`(16/160%/-2%)을 사용한다.

웹 Heading은 단계별 line-height 정책이 다르다. **Web-Heading-1~3은 140%**, **Web-Heading-4~6은 160%**가 기본이다. 즉 "Reading 160%는 본문 전용"이라는 일반 규칙은 웹 Heading-4 이하에는 적용되지 않으며, 토큰 값을 그대로 사용한다.

## Layout

기본 간격은 2, 4, 8, 12, 16, 20, 24, 32, 40px의 9단 scale을 사용한다(`spacing-02`~`spacing-40`). 컴포넌트 내부 간격, 리스트 간격, 섹션 간격은 이 scale 위에서만 조합한다. 좌우 margin, 브레이크포인트, artboard 같은 화면 단위 값은 별도의 `layout` 그룹에 둔다.

모바일과 태블릿의 좌우 기본 margin은 16px이다. iOS 기준 기본 artboard는 375×812pt, 최소 대응 범위는 320×568pt이다. AOS 기준 artboard는 360×800dp이며, 원본 가이드는 최소 해상도를 320×872dp로 기재하고 있다. 일반적인 안드로이드 단말의 최소 height와 어긋나는 표기이므로 본 시스템에서는 원본 값을 보존하되 실제 적용 시 320×720dp 이상에서 검증한다.

태블릿 구간은 674px부터 1440px까지를 기준으로 하며, 이미지는 필요할 경우 좌우 여백 없이 full-size로 사용할 수 있다.

레이아웃은 기능 단위의 명확한 그룹핑을 우선한다. 금융 정보 화면에서는 장식적 여백보다 스캔 가능한 정렬, 안정적인 행 높이, 예측 가능한 버튼 위치가 중요하다.

## Elevation & Depth

기본 깊이 표현은 강한 shadow보다 색상 면, 구분선, 여백, 상태 색으로 만든다. 밝은 배경 위에는 흰 surface와 `background-02` 면을 조합하고, 경계가 필요한 곳에 `line-03`을 사용한다.

Modal, Sheet, Popup처럼 화면 위에 올라오는 레이어는 radius 24px(`rounded.modal`)와 배경 dim, 명확한 닫기/확인 액션으로 hierarchy를 만든다. 단순 카드나 리스트 항목에는 무거운 그림자를 남발하지 않는다.

## Shapes

Radius는 4px 단위로 적용한다. 원본 Foundation Radius 토큰은 radius-04/08/12/16/24의 5단으로 정의되며, **radius-16이 카드·버튼 등 기본 컴포넌트의 Base 곡률**이다. design.md는 동일 값을 다음과 같이 매핑한다.

| Foundation | design.md key | value | 용도 |
|---|---|---|---|
| radius-04 | `rounded.sm` | 4px | 배지, 작은 강조, Text Field 기본 |
| radius-08 | `rounded.md` | 8px | 작은 버튼, 칩 내부 |
| radius-12 | `rounded.lg` | 12px | 중간 컨테이너 |
| radius-16 | `rounded.base` | 16px | 카드·버튼 기본 (Base) |
| radius-24 | `rounded.modal` | 24px | Modal·Sheet·Popup |

`rounded.none`(0)과 `rounded.full`(9999px)은 원본 토큰표에 없는 design.md 확장으로, none은 모서리가 없는 영역(이미지·라인) 표기, full은 chip·badge·avatar 같은 pill 형태에 사용한다.

작은 버튼과 칩 내부 요소에는 4~8px, 일반 박스 버튼과 카드에는 16px(Base), Modal·Sheet·Popup 같은 화면 위 레이어에는 24px을 사용한다. 한 화면 안에서 둥근 정도가 지나치게 섞이지 않도록 컴포넌트 유형별 radius를 유지한다.

아이콘은 24×24px Base를 기준으로 제작하고, line icon은 stroke 2px·corner radius 2px를 사용한다. 아이콘 스케일은 12/16/20/24/32/40의 6단계로 제공되며, 독립적으로 버튼처럼 동작하는 icon button은 최소 48×48px touch target을 확보한다.

## Components

**Icon:** 24×24 Base, stroke 2, corner radius 2. Naming은 `assetType-shapeName-direction-status-color` 패턴(예: `ico-chevron-down-disabled-gray`)을 따르고 모두 소문자·하이픈으로 표기한다. 형태보다 기능이 명확한 경우 기능 이름을 우선한다. 단독 사용 시 48×48 touch target을 확보한다.

**Avatar:** 기본 배경은 `background-02`, radius는 `rounded.full`. 사진이 없을 경우 initials 또는 placeholder icon으로 대체한다.

**Badge:** 짧은 카운트/상태 표시. `status-danger` 배경 + `text-01`을 기본 강조 배지(`badge`)로, 정보형 배지는 `background-02` + `text-04`(`badge-neutral`)를 사용한다. radius는 `rounded.full`, 타이포는 `caption-2`.

**Buttons:** Box button은 Primary, CTA, Secondary, Tertiary로 나뉜다. Primary와 CTA는 모두 Hecto Orange 기반이며, Primary는 표준 주요 행동(`button-primary`), CTA는 마케팅·결제·온보딩처럼 더 직접적인 유도 행동(`button-cta`)에 사용한다. Secondary는 대체/보조, Tertiary는 부가 행동이다. Large/Medium/Small/xSmall/xxSmall 크기를 지원하며 left/right icon 조합이 가능하다. hover/pressed/disabled는 토큰으로 정의된 상태 변종을 사용하고, Disabled 상태는 `disabled`(#D1D1D5)와 낮은 대비 텍스트로 조작 불가를 명확히 한다.

**Gradation and Grouped Buttons:** 검은 배경 위 강조 버튼처럼 특정 콘텐츠 강조 영역에서만 사용한다. 버튼 그룹은 중요 행동과 보조 행동의 비율을 명확히 유지하고, 여러 primary가 같은 강도로 경쟁하지 않게 한다.

**Text Button(`button-text`):** 아주 좁은 영역이나 텍스트 흐름 안의 보조 행동에만 사용한다. 텍스트 버튼은 `text-05` 또는 `primary-text-01` 컬러만 허용한다. primary 제출·결제·인증·삭제 같은 주요 행동에는 사용하지 않는다.

**Text Field:** enabled, focused, activated, error, focused-error, disabled, read-only 상태를 구분한다. Placeholder는 `text-06`, 일반 입력값은 `text-03`, helper text는 `text-05`, error helper와 border는 `red-50`을 사용한다. 값은 `status-danger`와 같더라도 입력 검증 맥락에서는 helper text와 함께 사용한다. disabled는 `disabled` 배경 + `text-06`, read-only는 `background-02` 배경 + `text-04`로 입력 가능 여부를 시각적으로 분리한다.

**Text Area, Picker, Search, Card Number:** 입력 계열은 Text Field 상태 체계를 공유한다. 긴 문장 입력은 Reading line-height를 사용하고, picker/search/card number처럼 포맷이 중요한 입력은 단위별 grouping과 clear/search/icon affordance를 유지한다.

**Selection Controls:** Checkbox, radio, switch 등 선택 컴포넌트는 selected/disabled/error의 색과 icon 상태를 분리한다. 선택 활성 색은 `hecto-orange-50`을 기본으로 하며 체크 마크/dot은 `text-01`(흰색)을 사용한다. error 상태는 `red-50` border로 표현한다.

**Navigation, Tabs, Pagination, Breadcrumbs, Stepper:** 현재 위치와 진행 상태를 먼저 보여준다. 활성 상태는 텍스트 굵기, indicator, primary/neutral 대비로 표현하고, 비활성 항목은 `text-05` 또는 `icon-05`를 사용한다. 탭 인디케이터(선·바)는 `hecto-orange-60`을 기본 활성 색으로 한다. 활성 텍스트 색은 `text-03`을 유지해 텍스트 강조와 인디케이터 색을 분리한다.

**Chip:** pill radius(`rounded.full`)와 neutral surface(`background-02` + `text-04`)를 기본으로 하고, 선택 상태는 `text-03` 배경 + `text-01` 텍스트로 강한 대비를 만든다.

**Notice / Toast / Tooltip:** 짧은 상태나 안내를 빠르게 전달하는 컴포넌트다. Notice는 정보의 심각도에 맞는 semantic color를 사용한다(중립=`background-02` 면, 오류=`red-50` border + `red-5` 면 등). Toast와 Tooltip은 본문을 길게 담지 않고 `text-03` 배경 + `text-01` 텍스트로 일관되게 표시한다. Tooltip은 보조 설명에 한정한다.

**Sheets / Bottom Sheet / Popup:** 중요한 선택 또는 확인이 필요한 경우 사용한다. 화면을 덮는 레이어에서는 24px radius(`rounded.modal`), 명확한 title, 하나의 primary action, 필요 시 secondary action을 유지한다.

**Progress Bar:** 트랙은 `background-02`, 진행 면은 `hecto-orange-50`을 기본으로 한다. 높이 4px, radius는 `rounded.full`. 무한 로딩은 진행 면을 좌→우 모션으로 반복한다.

**Loading, Empty Case, Error Case:** 사용자가 다음 행동을 이해할 수 있도록 상태 메시지와 복구 행동을 함께 제공한다. Error Case는 원인 설명보다 해결 가능한 행동을 우선한다.

## Do's and Don'ts

- 원시 팔레트 토큰보다 semantic color 토큰을 먼저 사용한다.
- 한 화면의 가장 중요한 행동에만 Hecto Orange 기반 `button-primary` 또는 `button-cta`를 사용한다.
- 정형 주요 행동(확인·완료·다음)에는 `button-primary`, 브랜드·결제·온보딩처럼 직접적인 유도 행동에는 `button-cta`를 사용한다.
- 탭·선택·프로그레스 등 상태 인디케이터에는 `hecto-orange-50/60`을 사용해 브랜드 톤을 유지한다.
- Pretendard와 정의된 type scale을 사용하고 임의의 폰트 크기를 만들지 않는다.
- 본문·약관·안내문처럼 긴 문장에는 line-height 160% reading 토큰을 사용한다.
- 모바일·태블릿 좌우 margin은 16px을 유지하되, 풀블리드 이미지/미디어 의도가 명확한 경우에만 0으로 둔다.
- 아이콘은 24×24 base 비율을 유지하고, 단독 사용 시 48×48 touch target을 확보한다.
- Text Field error 표기는 `red-50` border + helper 텍스트를 함께 사용하고, 색만으로 오류를 전달하지 않는다.
- 카드·기본 박스 버튼 radius는 `rounded.base`(16px)을 우선 적용하고, 작은 버튼·칩 내부에만 `rounded.sm/md`를 사용한다.
- 한 금융 워크플로 안에서 여러 강한 팔레트 색이 동시에 경쟁하지 않게 한다.
- 시각 위계의 1차 수단으로 무거운 그림자를 사용하지 않는다 — 면·선·간격·상태색으로 위계를 만든다.
- 4px 단위 radius scale 밖의 임의 radius를 만들지 않는다.
- 텍스트 버튼을 primary 제출·결제·인증·삭제 같은 주요 행동에 사용하지 않는다.
- 색만으로 검증/상태를 표현하지 않는다 — border/상태색은 helper 텍스트 또는 icon 피드백과 항상 함께 사용한다.
- 한 화면 안에서 box radius를 4·8·12·16·24로 동시에 섞지 않는다 — 컴포넌트 유형별 radius를 유지한다.
- `button-primary`와 `button-cta`를 같은 영역에서 동시에 강조 색으로 사용하지 않는다.
